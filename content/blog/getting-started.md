# Getting started

I want this blog to be a place for short notes while I work through my own documents, tools, and systems.

The goal is not polished essays every time. It is a lightweight record of decisions, tradeoffs, and progress that I can look back on later.

## What I want to track

- Ideas that are still forming
- Project notes that deserve more context than a commit message
- Small wins, rough edges, and lessons learned
- The path from messy document drafts to something useful

For now, keeping posts as markdown files feels right. It keeps the writing close to the code and makes publishing part of the same workflow as the rest of this site.
